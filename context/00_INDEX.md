# Flint — Project Context Index

> Welcome to the Flint project.

This directory contains the complete project context required to understand Flint before contributing to the codebase. Every developer, reviewer, and AI agent should begin here.

The purpose of these documents is to establish a shared understanding of the project, its architecture, engineering principles, current progress, and long-term vision.

# Documentation Principles

The documents in this directory focus on project knowledge rather than implementation details.

They answer questions such as:

- What are we building?
- Why are we building it?
- How is it designed?
- Why were certain decisions made?
- What is the current status?
- Where is the project heading?

Implementation details belong in the `/docs` directory and source code belongs in `/src`.

---

# Intended Audience

These documents are written for:

- Developers
- Project reviewers
- AI coding assistants
- Future contributors
- Hackathon judges (when appropriate)

Every reader should be able to understand the complete project before reading the source code.

---

# Documentation Standards

These documents should always remain:

- Accurate
- Up to date
- Concise
- Version controlled
- Easy to navigate

Whenever a major architectural or product decision is made, the relevant context document should be updated accordingly.
